# TxGNN 國際化可行性搜尋指南

## 目標

評估目標國家/地區是否具備建立 TxGNN 在地化應用的條件。

---

## 搜尋流程總覽

```
輸入：目標國家/地區
         ↓
    Phase 1：藥監局識別
         ↓
    Phase 2：開放資料搜尋
         ↓
    Phase 3：資料格式評估
         ↓
    Phase 4：欄位映射分析
         ↓
    Phase 5：可行性評分
         ↓
輸出：可行性報告
```

---

## Phase 1：藥監局識別

### 1.1 搜尋關鍵字

針對目標國家，搜尋以下關鍵字：

```
{Country} FDA
{Country} drug regulatory agency
{Country} pharmaceutical agency
{Country} medicine agency
{Country} 藥監局 / 藥品管理局
```

### 1.2 常見藥監局名稱模式

| 地區 | 常見命名 | 範例 |
|------|----------|------|
| 亞洲 | Ministry of Health + Drug/Pharmaceutical | 日本 PMDA、韓國 MFDS |
| 歐洲 | Medicines Agency | EMA、MHRA (英國) |
| 美洲 | FDA / Health Canada | US FDA、ANVISA (巴西) |
| 東南亞 | FDA / Drug Administration | Thai FDA、BPOM (印尼) |

### 1.3 產出

- [ ] 藥監局全名
- [ ] 藥監局縮寫
- [ ] 官方網站 URL

## Phase 2：開放資料搜尋

### 2.1 搜尋位置

依優先順序搜尋：

1. **藥監局官網**
   - 通常在「Open Data」「公開資料」「データ公開」等頁面
   - 或「Download」「下載專區」

2. **政府開放資料平台**
   ```
   {Country} open data portal
   {Country} government data
   data.gov.{country_code}
   ```

3. **國際藥品資料庫**
   - WHO Drug Information
   - DrugBank (已有資料)

### 2.2 搜尋關鍵字

```
{Agency} drug database download
{Agency} approved drugs list
{Agency} pharmaceutical products data
{Country} registered medicines data
{Country} 藥品許可證 資料集
```

### 2.3 產出

- [ ] 資料下載 URL
- [ ] 資料格式（JSON/CSV/XML/ZIP）
- [ ] 更新頻率
- [ ] 授權條款（是否允許使用）

## Phase 3：資料格式評估

### 3.1 下載樣本資料

取得資料後，檢查：

```bash
# 檔案大小
ls -lh {data_file}

# 如果是 JSON
head -100 {data_file}.json | python -m json.tool

# 如果是 CSV
head -10 {data_file}.csv

# 如果是 ZIP
unzip -l {data_file}.zip
```

### 3.2 評估項目

| 項目 | 理想狀態 | 可接受 | 不可行 |
|------|----------|--------|--------|
| 格式 | JSON/CSV | XML | PDF/HTML only |
| 編碼 | UTF-8 | 可轉換 | 無法識別 |
| 結構 | 扁平表格 | 巢狀但可解析 | 非結構化 |
| 資料量 | >10,000 筆 | >1,000 筆 | <100 筆 |

### 3.3 產出

- [ ] 資料格式確認
- [ ] 編碼確認
- [ ] 總筆數
- [ ] 樣本資料（5-10 筆）

## Phase 4：欄位映射分析

### 4.1 必要欄位檢查

對照樣本資料，確認以下欄位是否存在：

| 標準欄位 | 可能的當地名稱 | 存在？ | 品質 |
|----------|----------------|--------|------|
| license_id | 許可證號、承認番号、허가번호 | | |
| brand_name | 品名、販売名、제품명 | | |
| ingredients | 成分、有効成分、성분 | | |
| indication | 適應症、効能効果、효능효과 | | |
| status | 狀態、承認状況、허가상태 | | |

### 4.2 成分欄位分析（關鍵）

檢查成分欄位：

```
✓ 理想：英文學名
  例：METFORMIN HYDROCHLORIDE

○ 可接受：當地語言 + 英文對照
  例：メトホルミン塩酸塩 (Metformin Hydrochloride)

△ 需額外處理：僅當地語言
  例：メトホルミン塩酸塩
  → 需建立翻譯對照表

✗ 不可行：僅品牌名、無成分資訊
```

### 4.3 適應症欄位分析（關鍵）

檢查適應症欄位：

```
✓ 理想：結構化疾病名稱
  例：["diabetes mellitus", "hypertension"]

○ 可接受：當地語言文字描述
  例：糖尿病、高血壓の治療
  → 需建立 DISEASE_DICT

△ 需額外處理：長篇敘述
  例：本剤は血糖降下作用を有し、2型糖尿病患者の...
  → 需 NLP 提取

✗ 不可行：無適應症資訊
```

### 4.4 產出

- [ ] 欄位對照表
- [ ] 成分欄位品質評估
- [ ] 適應症欄位品質評估
- [ ] 預估映射難度

## Phase 5：可行性評分

### 5.1 評分標準

| 項目 | 權重 | 評分 (0-10) |
|------|------|-------------|
| 資料可取得性 | 20% | |
| 資料格式友善度 | 15% | |
| 成分欄位品質 | 25% | |
| 適應症欄位品質 | 25% | |
| 資料量充足度 | 10% | |
| 授權條款 | 5% | |
| **總分** | 100% | |

### 5.2 評分說明

**資料可取得性 (0-10)**
- 10: 公開下載、無需註冊
- 7: 需註冊但免費
- 4: 需申請審核
- 0: 無法取得

**資料格式友善度 (0-10)**
- 10: JSON/CSV、UTF-8、扁平結構
- 7: XML 或需簡單轉換
- 4: 需複雜解析
- 0: 非結構化

**成分欄位品質 (0-10)**
- 10: 英文學名
- 7: 當地語言 + 英文對照
- 4: 僅當地語言但可翻譯
- 0: 無成分資訊

**適應症欄位品質 (0-10)**
- 10: 結構化疾病名稱
- 7: 當地語言短句
- 4: 長篇敘述需 NLP
- 0: 無適應症資訊

**資料量充足度 (0-10)**
- 10: >50,000 筆
- 7: 10,000-50,000 筆
- 4: 1,000-10,000 筆
- 0: <1,000 筆

**授權條款 (0-10)**
- 10: 開放授權、可商用
- 7: 開放授權、僅學術
- 4: 需署名或有限制
- 0: 禁止使用

### 5.3 可行性判定

| 總分 | 判定 | 建議 |
|------|------|------|
| 8.0+ | 高度可行 | 立即開始 |
| 6.0-7.9 | 可行 | 需額外處理 |
| 4.0-5.9 | 有挑戰 | 需評估成本 |
| <4.0 | 不建議 | 等待資料改善 |

## 快速評估清單

對於新的目標地區，快速回答以下問題：

1. [ ] 藥監局是否有公開藥品資料庫？
2. [ ] 資料是否可下載（非僅網頁查詢）？
3. [ ] 資料是否包含成分資訊？
4. [ ] 成分是否有英文名稱？
5. [ ] 資料是否包含適應症？
6. [ ] 資料量是否超過 1,000 筆？

**若以上皆為「是」，則值得進入完整評估流程。**
